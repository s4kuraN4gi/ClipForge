# ClipForge 辛口審査結果 (2026-02-23 第2回)

## 審査エージェント
1. 収益化/プライシング
2. セキュリティ/バックエンド
3. プロダクト/UX
4. 運用/ローンチ

## BLOCKER（リリース前に必須）

### 1. 特定商取引法・利用規約・プライバシーポリシーが未実装 [Legal]
- footer のリンクが全て `#`（実ページなし）
- 日本で課金サービスを運営するなら法的に必須
- `/terms`, `/privacy`, `/tokushoho` ページを作成

### 2. Seedance API 本番化の確認 [運用]
- `.env.example` に `SEEDANCE_USE_MOCK=true` がデフォルト
- `BYTEPLUS_API_KEY` 未設定時に自動で mock fallback（silent failure）
- 本番で課金ユーザーにモック動画を返すリスク

### 3. サンプル動画が 0-byte [UX]
- `public/samples/videos/*.mp4` が全て空ファイル
- サンプル体験フローが完全に壊れている

### 4. in-memory レート制限が Vercel serverless で無効 [セキュリティ]
- Map は各インスタンスで別 → 制限なしと同じ
- Upstash Redis 等に移行必要

### 5. エラー監視なし（Sentry 等） [運用]
- console.error のみ → アラートなし
- Webhook 失敗や API エラーが検知されない

### 6. Vercel Function タイムアウト設定なし [運用]
- デフォルト 60s だが Seedance API は数分かかる可能性
- vercel.json で maxDuration 設定が必要

## HIGH（リリース後1週間以内）

- 二重課金防止（active subscription チェックなし）
- 署名付きURL有効期限が24時間（長すぎる→1時間に）
- ファイルアップロードの magic bytes チェックなし
- IP アドレス取得ロジック（x-forwarded-for の最初の値を信頼→最後の値が正）
- CSP に unsafe-inline / unsafe-eval
- Webhook べき等性の完全実装（event_id 記録）
- 外部 URL フェッチの SSRF 対策
- DB インデックス追加（generated_videos.status）
- Storage バックアップ戦略
- チャーン対策（解約防止メール等）
- アップセル導線（Starter→Business）

## MEDIUM（リリース後2週間以内）

- 料金プランの利益率改善検討（Business 38%は低い）
- 無料プラン3本が多すぎる可能性（1本/月 or テンプレ1種制限）

- モバイル 375px 以下でギャラリー破綻
- アクセシビリティ改善
